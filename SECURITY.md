# Security Policy

NewsVideo は個人開発の `OSS Public Beta` です。セキュリティ報告の受付は行いますが、対応は `best effort` で、SLA や対応期限の保証はありません。

## 対応対象

現時点では、次のような問題をセキュリティ報告として扱います。

- API キーやローカルデータの意図しない漏えい
- 任意コード実行や任意ファイル書き込みにつながる問題
- プロジェクト外のファイルにアクセスできてしまう問題
- 配布物の改ざんやバンドル内容の整合性に関わる問題

次のものは、原則として通常のバグや制約の範囲です。

- 生成品質のばらつき
- 外部 API の障害、レート制限、クォータ超過
- `Windows` / `Linux` / `Intel Mac` など非対応環境での問題
- 既知の Public Beta 制約に含まれる挙動

## 報告方法

### 公開して問題ない内容

- [GitHub Issues](https://github.com/aoyaman41/NewsVideo/issues/new/choose) から報告してください

### 公開すると危険な内容

- API キー、トークン、個人情報、再現用の秘密情報は公開 Issue に貼らないでください
- GitHub の private vulnerability reporting が有効なら、そちらを使ってください
- 使えない場合は、公開 Issue を作る前にリポジトリ所有者へ非公開で連絡できる手段があるか確認してください

## 報告時にほしい情報

- 何が起きるか
- どの環境で起きるか
- 再現手順
- 影響範囲
- 秘密情報を含まない最小の再現例

## 期待値

- 受領確認や修正は `best effort` です
- すべての報告に対して修正を約束するものではありません
- バグ報奨金制度はありません
