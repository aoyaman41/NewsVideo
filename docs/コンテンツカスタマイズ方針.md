# コンテンツカスタマイズ方針

このドキュメントは、OSS 向け一般化で採用する `プリセット中心 + 一部だけカスタム上書き` 方針をまとめたものです。自由入力を増やしすぎて生成品質やサポート負荷を崩さないため、設定ごとに露出境界を決めています。

実装上のソースオブトゥルースは `shared/project/contentCustomization.ts` です。

## 1. 分類

### プリセットのみ

- 画像スタイル
- レイアウト方針

考え方:

- 表現の核は preset で守ります
- 自由入力の style prompt や raw 指示は通常 UI に出しません

### プリセット + カスタム上書き

- 締め文
- 締めカード
- TTS 話法
- CTA
- 冒頭文
- ブランド名
- 出典表示方針

考え方:

- 既定の表現は preset に持たせます
- 案件差分が大きい文言だけ、`なし` や任意テキストで上書きします

### advanced only

- 画像 prompt 上書き
- TTS prompt 上書き
- レイアウトテンプレート
- ブランド描画ルール

考え方:

- 生成品質や UI 複雑化への影響が大きいものは advanced に閉じます
- Phase 2 以降で実装するとしても、初期 UI には出しません

### 通常設定

- パート数
- 1パートの目安秒数
- アスペクト比
- 解像度
- 音声ボイス
- オープニング / エンディング素材

考え方:

- これはスタイル preset ではなく、出力条件に直接効く通常設定として扱います

## 2. UI 露出方針

- `main`: いまの主要フローで頻繁に触る設定。`配信スタイル`、`締め文`、`音声の話し方`、`パート数`、`1パートの目安秒数` など
- `details`: 利用シーンによって触る設定。画像スタイル、アスペクト比、ブランド名など
- `advanced`: 初期 UI には出さず、将来的に必要なら詳細設定として隔離する設定

## 3. 現在の適用

- Phase 1 で既に露出しているのは `配信スタイル`、`締め文`、`1パートの目安秒数` です
- Phase 2 で `画像スタイル` と `音声の話し方` が追加され、`音声の話し方` は短い補足上書きも許容します
- 動画画面の `締めカード`、`CTA`、`出典表示方針` は project 単位の outro テンプレートとして扱い、app settings の opening / ending 動画素材とは分離します
- 後続 issue はこの分類に従って、`main` に置くか `details/advanced` に送るかを決めます
