# FAQとトラブルシュート

このドキュメントは、NewsVideo の `Public Beta` を初めて使うときに詰まりやすい項目を、自己解決しやすい順に整理したものです。

## アプリが起動しない

### 症状

- 初回起動で macOS の警告が出る
- `Apple could not verify ...` や `developer cannot be verified` と表示される

### まず確認すること

- 配布版は `macOS / Apple Silicon (arm64)` 向けです
- 現在の Public Beta 配布物は `Developer ID` 署名 / `notarization` なしです

### 対処

- まず [macOSインストールとGatekeeper](./macOSインストールとGatekeeper.md) の手順を確認してください
- `Applications` にコピーしたうえで、`control`-click または右クリックから `開く` を試してください
- 管理対象 Mac や厳しいセキュリティポリシーの端末では実行できない場合があります

## API キーの接続テストに失敗する

### まず確認すること

- キーの貼り付けミスがないか
- 対象サービス側で API 利用が有効になっているか
- ネットワークや VPN、社内プロキシが接続を止めていないか

### 対処

- `OpenAI` と `Google AI` のどちらを使う構成かを [クイックスタート](./クイックスタート.md) で確認してください
- キーを保存し直したうえで、もう一度 `接続テスト` を実行してください
- `Google TTS` は現在の標準フローでは必須ではありません

## 生成が途中で失敗する

### よくある原因

- モデル設定を変えた直後で、自動保存が終わる前に生成を始めた
- 現在のモデル構成に必要な API キーが足りない
- プロバイダ側のクォータ超過、レート制限、障害が起きている

### 対処

- 設定画面の状態が `保存済み` になってから再実行してください
- `Google AI` だけで始めたい場合は、文章生成モデルを両方とも `Gemini 3.1 Pro` に変更してください
- OpenAI 系モデルを使うなら `OpenAI` キー、画像生成や Gemini TTS には `Google AI` キーが必要です
- 失敗が続く場合は、エラーメッセージを控えて Issue を作成してください

## 課金は誰負担か

- API 利用料は NewsVideo ではなく、ユーザー自身の OpenAI / Google AI 契約に対して直接発生します
- ローカルアプリの利用それ自体ではなく、生成時の API 呼び出しに対して課金されます
- 詳細は各プロバイダの料金体系を確認してください

## 対応環境と非対応環境はどこまでか

### 公式に案内している範囲

- `macOS / Apple Silicon (arm64)` の配布版
- `macOS` 上でのソースビルド

### 現在サポートしないもの

- `Windows`
- `Linux`
- `Intel Mac`
- 商用 SaaS のような継続サポートや SLA

詳細は [対応環境と制約](./対応環境と制約.md) を確認してください。

## それでも解決しない場合

- まず `README`、[クイックスタート](./クイックスタート.md)、[対応環境と制約](./対応環境と制約.md) を見直してください
- そのうえで再現条件がある場合は、[Issue 作成画面](https://github.com/aoyaman41/NewsVideo/issues/new/choose) から報告してください
- 個人開発の `OSS Public Beta` のため、サポートは `best effort` です
