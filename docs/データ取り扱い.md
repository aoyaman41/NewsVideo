# データ取り扱い

このドキュメントは、NewsVideo の Public Beta で「何がローカルに保存され」「何が外部 API に送られるか」を、現行実装ベースで整理したものです。

## 先に結論

- API キーはプロジェクトファイルとは別に保存されます
- 記事本文、台本、画像プロンプト、音声生成テキストは、生成に使う外部 API へ送信されます
- 画像、音声、動画、プロジェクトデータはローカルに保存されます
- 現時点では専用のテレメトリ送信やログ収集はありません

## ローカルに保存されるもの

### API キー

- API キーは Electron の `safeStorage` を使って暗号化し、`secrets.enc` に保存します
- プロジェクトごとの `.newsproj` ディレクトリには保存しません
- `safeStorage` が使えない環境では保存できません

実装根拠:

- `electron/ipc/settings.ts`
- `electron/ipc/ai.ts`
- `electron/ipc/image.ts`
- `electron/ipc/tts.ts`

### 通常設定

- モデル選択、解像度、TTS 設定などの通常設定は `settings.json` に保存します
- ここには API キー本体は入りません

### プロジェクトデータ

各プロジェクトは `projects/*.newsproj/` 配下に保存されます。代表的なファイルは次のとおりです。

- `project.json`: プロジェクト名、更新日時、サムネイル、最新動画パスなど
- `article.json`: 記事タイトル、出典、本文、取り込み画像の参照
- `parts.json`: パート構成、台本、割り当て画像、音声参照
- `prompts.json`: 画像プロンプト
- `images.json`: 生成 / 取込画像のメタデータ
- `audio.json`: 生成音声のメタデータ
- `usage.json`: モデル名、トークン数、画像枚数などの使用量記録
- `images/`, `audio/`, `output/`: 実ファイルとしての画像、音声、動画

補足:

- これらのプロジェクトファイルには記事本文や生成台本がそのまま入ります
- プロジェクトフォルダを共有すれば、その中身も一緒に共有されます

## 外部 API に送られるもの

### 接続テスト

`設定 > APIキー > 接続テスト` は、対象サービスの疎通確認用エンドポイントにアクセスします。

- OpenAI: `https://api.openai.com/v1/models`
- Google AI: `https://generativelanguage.googleapis.com/v1beta/models`
- Google TTS: `https://texttospeech.googleapis.com/v1/voices`

このとき送るもの:

- API キー

### テキスト生成

スクリプト生成、画像プロンプト生成、コメント反映は、選択中のテキストモデルに応じて `OpenAI` または `Google AI` を使います。

外部に送られうるもの:

- 記事タイトル
- 出典
- 記事本文
- パートタイトル
- パート要約
- 台本本文
- ユーザーが入力したコメント
- 選択中モデルに必要な生成パラメータ

### 画像生成

画像生成は `Google AI` を使います。

外部に送るもの:

- 画像プロンプト
- ネガティブプロンプト
- アスペクト比
- 画像モデル
- 解像度に応じた画像サイズ設定

### 音声生成

音声生成エンジンごとに送信先が変わります。

- `gemini_tts`: `Google AI`
- `google_tts`: Google Cloud Text-to-Speech
- `macos_tts`: 外部送信なし

外部に送られうるもの:

- 読み上げる台本テキスト
- 分割後のセグメント
- 音声設定
  - voice 名
  - language code
  - speaking rate
  - pitch

### 外部送信しない処理

以下は現行実装ではローカル処理です。

- プロジェクト作成、保存、再読込
- 画像 / 音声 / 動画ファイルの配置
- 動画プレビュー生成
- 最終動画の書き出し
- Finder での表示

## ログの扱い

### 現在あるもの

- Main プロセスには簡易 logger があります
- 既定ログレベルは `development` で `info`、それ以外は `warn`
- ログ出力先はコンソール標準出力 / 標準エラーです

### 現在ないもの

- 専用の外部ログ収集サービスへの送信
- 利用分析やテレメトリの自動送信
- API キーを平文で出力する専用ログ

### 現在のログで残すもの

現行コードで残しているのは主に次のようなメタデータです。

- `projectId`
- `promptId`
- 文字数
- 画像枚数
- 候補数
- 例外オブジェクト

現行コードでは、画像生成ログにプロンプト本文そのものは出していません。

## 境界のまとめ

ローカルに残るもの:

- API キーの暗号化保存
- 設定
- 記事本文
- 台本
- 画像プロンプト
- 画像 / 音声 / 動画ファイル
- 使用量メタデータ

外部に送られるもの:

- 生成に必要な記事本文、台本、プロンプト、音声テキスト
- 接続テストや生成処理に必要な API キー
