# 公開リリース手順

このドキュメントは、NewsVideo の `Public Beta` を GitHub Releases に載せるための現行手順を整理したものです。基準ブランチは `main`、配布対象は `macOS / Apple Silicon (arm64)` のみです。

## リリース方針

- 配布チャネルは GitHub Releases
- リリース元ブランチは `main` のみ
- タグは `vX.Y.Z` 形式
- リリース名は `NewsVideo vX.Y.Z Public Beta`
- 既定では GitHub の `prerelease` フラグを付けない
- 理由: README やクイックスタートで使っている `releases/latest` をそのまま有効に保つため
- 個人開発の `OSS Public Beta` として公開し、サポートは `best effort`
- 個別環境ごとの無償セットアップ支援や運用代行はしない

## 署名と notarization

- 現時点では `Developer ID` 署名を整備していません
- 現時点では `notarization` も行っていません
- 配布物は `Public Beta / best effort` 前提です

この扱いは [docs/対応環境と制約.md](./対応環境と制約.md) と揃えて運用します。

## GitHub Actions の既定フロー

既定運用は `.github/workflows/release-public-beta.yml` を手動実行する方法です。

### 実行前チェック

- `main` に対象変更がマージ済み
- `package.json` の `version` が今回の公開版に更新済み
- 直近の CI が通っている
- [リリース前スモークテスト](./リリース前スモークテスト.md) を完了している
- README、クイックスタート、制約ドキュメントが現行挙動と一致している
- 公開用の画面素材を更新した場合は `npm run docs:public-assets` を実行済み

### 実行手順

1. GitHub Actions で `Release Public Beta` workflow を開く
2. 実行対象の ref は `main` を選ぶ
3. `version` に `package.json` と同じ値を入力する
4. `prerelease` は原則 `false` のままにする
5. workflow 完了後、GitHub Releases に新しい公開版ができていることを確認する

### workflow がやること

- `npm ci`
- `npm run lint`
- `npm run typecheck`
- `npm test`
- `npm run build:release:mac`
- 成果物の存在確認
- リリースノートの雛形生成
- workflow artifact の保存
- GitHub Release の作成 / 更新

## 成果物の型

workflow と手動公開の両方で、以下を標準成果物とします。

- `NewsVideo-X.Y.Z-arm64.dmg`
- `NewsVideo-X.Y.Z-arm64.dmg.blockmap`
- `NewsVideo-X.Y.Z-arm64-mac.zip`
- `NewsVideo-X.Y.Z-arm64-mac.zip.blockmap`
- `latest-mac.yml`

補足:

- スクリーンショットや GIF はリポジトリの `.github/assets/public-beta/` で管理します
- Releases に添付したい場合も、同じ素材を再利用します

## リリースノートの型

最低限、以下を毎回含めます。

- `Public Beta` であること
- 個人開発 / `best effort` サポートであること
- 対応環境が `macOS / Apple Silicon (arm64)` のみであること
- `Windows` / `Linux` / `Intel Mac` を現時点でサポートしないこと
- `Developer ID` 署名なし / `notarization` なしであること
- Gatekeeper 警告が出る可能性と回避手順へのリンク
- 個別環境の無償セットアップ支援は対象外であること
- 外部 API 障害やモデル変更への即時対応を保証しないこと
- クイックスタートと制約ドキュメントへのリンク
- その公開版がどの commit から作られたか

## 手動 fallback 手順

GitHub Actions が使えない場合は、ローカルの macOS 環境で以下を実行します。

```bash
npm ci
npm run lint
npm run typecheck
npm test
npm run build:release:mac
```

その後、GitHub Releases で以下を手動登録します。

1. タグ `vX.Y.Z` を作成する
2. タイトルを `NewsVideo vX.Y.Z Public Beta` にする
3. 本文に署名 / notarization 未対応を明記する
4. `release/` 配下の標準成果物をアップロードする

## 変更時の更新ポイント

以下が変わったら、このドキュメントと workflow を同じ PR で更新します。

- 対応 OS / CPU
- リリース成果物の名前
- 署名 / notarization の有無
- Actions の入力項目
- README の `releases/latest` へのリンク方針
