# 現行実装ガイド

このドキュメントは、2026-04-26 時点の `main` 実装に合わせた運用ガイドです。

## 1. 実行レイヤー

- `src/**` は Renderer（React UI）。
- `electron/main.ts` は Electron Main プロセス。
- `electron/preload.ts` は `window.electronAPI` のブリッジ層。
- `electron/ipc/*.ts` は Main 側の IPC ハンドラー実装。

## 2. 主要フロー

- プロジェクト保存・読込: `project:*`（`electron/ipc/project.ts`）
- 設定・APIキー: `settings:*`（`electron/ipc/settings.ts`）
- スクリプト生成: `ai:generateScript`（`electron/ipc/ai.ts`）
- 画像プロンプト生成: `ai:generateImagePrompts`, `ai:generateImagePromptForTarget`（`electron/ipc/ai.ts`）
- 画像生成: `image:generate`, `image:generateBatch`（`electron/ipc/image.ts`）
- 音声生成: `tts:generate`, `tts:generateBatch`, `tts:getVoices`（`electron/ipc/tts.ts`）
- 動画出力: `video:render`, `video:preview`, `video:cancelRender`（`electron/ipc/video.ts`）

## 3. モデル利用（現行）

- テキスト生成（スクリプト/画像プロンプト）:
- `gpt-5.5` / `gpt-5.4` / `gpt-5.2` / `gemini-3.1-pro`（設定で選択）
- `gemini-3.1-pro` は API 呼び出し時に `gemini-3.1-pro-preview` を使います。
- 画像生成:
- `gemini-3.1-flash-image-preview`（既定）/ `gemini-3-pro-image-preview`
- TTS:
- `google_tts` / `gemini_tts` / `macos_tts`
- Gemini TTS のモデルは `gemini-3.1-flash-tts-preview`（既定）/ `gemini-2.5-pro-preview-tts` / `gemini-2.5-flash-preview-tts` から選択できます。

## 4. 配信スタイルと締め文

- 記事画面では `project.presentationProfile` を持ち、プロジェクト単位で台本生成方針を保持します。
- 現在の preset は `news`、`explain`、`report`、`short` の4種類です。
- 各 preset は `tone`、`targetDurationPerPartSec`、既定の締め文を持ちます。
- 締め文は `preset / none / custom` で切り替えます。
- 音声の話し方は `news`、`explain`、`casual`、`promo` の preset で選び、短い補足文だけ上書きできます。
- `ai:generateScript` は `tone`、`targetDurationPerPartSec`、`closingLine` を受け取り、通常生成と記事画面の自動生成で同じ設定を使います。
- `presentationProfile` を持たない既存プロジェクトは `news + preset closing` として読み込みます。
- 一般化の境界定義は `shared/project/contentCustomization.ts` と [コンテンツカスタマイズ方針](./コンテンツカスタマイズ方針.md) を参照してください。

## 4.1 音声スタイルと app settings の分離

- `ttsEngine` と `ttsVoice` は app settings 側の既定値です。
- `presentationProfile.ttsNarrationStylePreset` と `ttsNarrationStyleNote` は project 単位の話し方設定です。
- Gemini TTS ではこの話し方 preset を instruction prompt として適用します。

## 4.2 動画 outro と app settings の分離

- `presentationProfile.closingCardEnabled`、`closingCardHeadline`、`closingCardCtaText`、`sourceDisplayMode`、`sourceDisplayText` は project 単位の outro 設定です。
- 動画書き出しでは `opening video -> 各パート -> closing card -> ending video` の順で連結します。
- `settings.openingVideoPath` と `settings.endingVideoPath` は app 全体の共通素材パスで、動画画面では今回の書き出しに含めるかだけ切り替えます。
- closing card 内の出典行は `auto / hidden / custom` で切り替え、`auto` のときだけ記事の `source` を `出典: ...` 形式で表示します。
- `closing card` 系フィールドがない既存 project は互換性のため `closingCardEnabled = false` で読み込みます。

## 5. 画像スタイル方針

- 画像スタイルは `infographic`、`editorial`、`minimal`、`social`、`textRich`、`dataCard` の preset に限定します。
- project の `presentationProfile.imageStylePreset` と `presentationProfile.aspectRatio` を、画像プロンプト生成と画像生成の両方で使います。
- project の `presentationProfile.styleReferenceImageIds` は、画像画面で選んだ最大3枚のスライドサンプルを保持します。
- `presentationProfile.styleReferenceNote` は、参照サンプルから特に合わせたい点だけを短く補足します。
- 新規 project と、`aspectRatio` がない既存 project は app settings の `defaultAspectRatio` を初期値として使います。
- 画像 style の raw prompt 上書きはまだ advanced 扱いで、通常 UI には出していません。

## 6. 画像プロンプト生成方針（重要）

- 1回の長文抽出ではなく「1パートごとの抽出」を採用しています。
- バッチ時は最大10並列で抽出し、最終プロンプトをパート単位で組み立てます。
- 抽出結果は `visualCopy` と `layoutPlan` を優先し、見出し・要点・数値・オブジェクト配置を構造化して保存できます。
- 単体再生成も同じ抽出ロジックを再利用します。
- 最終プロンプトのレイアウト指示は `aspectRatio` に応じて横長 / 正方形 / 縦長へ切り替えます。
- スタイル参照画像が指定されている場合は、画像生成時に参照画像として API に渡し、プロンプト上でも「内容はコピーせずデザイン文法だけを合わせる」よう指示します。
- 画像生成バッチは最大3並列で実行し、一部失敗した場合は成功分を保存したうえで失敗パートを画面に表示します。

## 7. ドキュメントと実装の優先順位

- 仕様が食い違う場合は、必ず実装コードを正として扱ってください。
- docs は実装の説明であり、実装を拘束する古い設計書としては扱いません。

## 8. 開発時フラグ（Electron Main）

- `NEWSVIDEO_OPEN_DEVTOOLS=1` のときだけ DevTools を自動起動します（`electron/main.ts`）。
- `NEWSVIDEO_LOG_LEVEL` で Main 側ログレベルを制御できます（`error|warn|info|debug`）。
- ログ出力は `electron/utils/logger.ts` 経由に統一し、画像生成時はプロンプト本文を出力しません。
