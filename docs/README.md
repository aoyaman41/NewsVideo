# docs ディレクトリ運用ルール

このディレクトリは「現在の実装を正しく伝える最小ドキュメント」のみを置きます。

## 運用ポリシー

- ソースオブトゥルースは常に実装コードです。
- 実装変更時は、同じPR/コミットで docs も更新します。
- 予定・構想・理想像は原則書きません。現行挙動だけを書きます。
- 実装とズレたドキュメントは修正ではなく削除を優先します。
- 仕様根拠は「ファイル名・関数名・IPC名」で追えるように記述します。
