# クイックスタート

このドキュメントは、2026-03-20 時点の `main` 実装を前提に、初見ユーザーが最短で 1 本目の動画を書き出すための手順をまとめたものです。

## 1. 先に結論

- 既定設定のまま始めるなら `OpenAI` と `Google AI` の両方の API キーが必要です
- `Google AI` だけで始めたい場合は、設定画面で文章生成モデルを両方とも `Gemini 3.1 Pro` に変更してください
- 現在の標準フローでは追加の TTS 専用 API キーは不要です。既定の音声エンジンは `Gemini TTS` です
- API 利用料は NewsVideo ではなく、各プロバイダの契約に対して直接発生します

## 2. どのキーが必要か

| 使い方 | 必要なキー | 補足 |
| --- | --- | --- |
| 既定設定のまま使う | `OpenAI` + `Google AI` | 既定のスクリプト生成モデルと画像プロンプト生成モデルは `GPT-5.2`、画像生成と既定の音声生成は Google 側 API を使います。 |
| Google AI だけで始める | `Google AI` | `設定 > 音声 > スクリプト生成モデル` と `設定 > 画像 > 画像プロンプト生成モデル` を両方とも `Gemini 3.1 Pro` に変更してください。 |
| OpenAI だけで始める | 不可 | 画像生成は `Google AI` が必要です。 |

API キー取得先:

- OpenAI: [platform.openai.com](https://platform.openai.com/)
- Google AI: [aistudio.google.com](https://aistudio.google.com/)

## 3. インストール

### 配布版

1. [GitHub Releases](https://github.com/aoyaman41/NewsVideo/releases/latest) から最新版を取得します
2. `.dmg` を開いて `NewsVideo.app` を `Applications` にコピーします
3. アプリを起動します

補足:

- 現在の Public Beta 配布物は `Developer ID` 署名 / `notarization` なしです
- これは未着手ではなく、個人開発 OSS の現状では年額コストと運用負荷に見合わないため、意図的に見送っているものです
- 初回起動時に macOS の Gatekeeper 警告が出る場合があります
- 警告の意味と回避手順は [macOSインストールとGatekeeper](./macOSインストールとGatekeeper.md) を確認してください

### 開発版

```bash
npm install
npm run dev
```

必要環境は `macOS` と `Node.js ^20.19.0 || >=22.12.0` です。

## 4. 最初の設定

1. アプリ右上またはサイドバーから `設定` を開きます
2. `APIキー` タブで必要なキーを入力して `保存` を押します
3. そのまま `接続テスト` を押して成功を確認します
4. `Google AI` だけで始める場合は、次の 2 か所も変更します

   - `音声` タブの `スクリプト生成モデル` を `Gemini 3.1 Pro` にする
   - `画像` タブの `画像プロンプト生成モデル` を `Gemini 3.1 Pro` にする

5. 画面上部の設定ステータスが `保存済み` になるまで待ちます

補足:

- `Google AI` は画像生成、Gemini 系の文章生成、Gemini TTS に使います
- `OpenAI` は OpenAI 系の文章生成モデルを選んだときだけ必要です
- 現在のアプリは `Gemini TTS` を既定の音声エンジンとして使用します

## 5. 1本目の動画を作る

1. `プロジェクト` 画面で新規プロジェクトを作成します
2. `記事` 画面でタイトル、出典、本文を入力します
3. すぐ試したい場合は [サンプル記事](./サンプル記事.md) を使います
4. 必要に応じて `配信スタイル`、`締め文`、`1パートの目安秒数` を調整します
5. `自動生成` カードから `記事から動画まで自動生成` を実行します
6. 処理が終わったら `動画` 画面でプレビューと書き出しを行います

手動で進めたい場合は `記事 -> スクリプト -> 画像 -> 音声 -> 動画` の順に各画面を使ってください。

補足:

- `配信スタイル` は `ニュース / 解説 / 報告 / ショート` の 4 種類です
- `締め文` は `既定 / なし / カスタム` から選べます
- `1パートの目安秒数` は台本の密度に影響します
- 既存プロジェクトにこの設定がない場合は `ニュース + 既定の締め文` として扱われます

## 6. よくあるつまずき

### `OpenAI APIキーが設定されていません` と出る

- 原因: `GPT-5.2` または `GPT-5.4` を使う設定のまま、`OpenAI` キーを入れていません
- 対処: `OpenAI` キーを保存するか、`スクリプト生成モデル` と `画像プロンプト生成モデル` を両方とも `Gemini 3.1 Pro` に切り替えます

### `Google AI APIキーが設定されていません` と出る

- 原因: 画像生成、Gemini 系文章生成、Gemini TTS のいずれかで `Google AI` が必要です
- 対処: `Google AI` キーを保存して `接続テスト` を通します

### 接続テストは成功したのに生成時に失敗する

- 原因: モデル変更後の自動保存が終わる前に生成を開始した可能性があります
- 対処: 設定画面のステータスが `保存済み` になっていることを確認してから再実行します

### `Google TTS APIキーが設定されていません` と出る

- 原因: 既定外の TTS フローや過去データで `google_tts` が参照されている可能性があります
- 対処: まずは現在の標準フローどおり `Gemini TTS` 前提で試してください
