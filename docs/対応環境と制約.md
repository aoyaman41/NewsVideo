# 対応環境と制約

このドキュメントは、2026-03-20 時点の `main` 実装と現在の配布物を基準に、NewsVideo の公式サポート範囲と Public Beta の制約を整理したものです。

## 1. 公式サポート範囲

### 配布版

| 項目 | 現在の扱い |
| --- | --- |
| OS | `macOS` |
| CPU | `Apple Silicon (arm64)` |
| 配布形式 | GitHub Releases の `.dmg` / `.zip` |
| サポートレベル | `Public Beta / best effort` |

現行の公開アセットは `NewsVideo-2.0.1-arm64.dmg` と `NewsVideo-2.0.1-arm64-mac.zip` です。`Intel Mac` 向け、`Windows` 向け、`Linux` 向けの公式配布物は現在ありません。

### ソースビルド

| 項目 | 現在の扱い |
| --- | --- |
| OS | `macOS` |
| Node.js | `^20.19.0` または `>=22.12.0` |
| 起動方法 | `npm install` → `npm run dev` |
| サポートレベル | `best effort` |

ソースビルド前提でも、現時点で積極的に案内しているのは `macOS` だけです。

## 2. 現在サポートしないもの

- `Intel Mac` 向けの公式配布物
- `Windows` 向けの公式配布物
- `Linux` 向けの公式配布物
- 非 macOS 環境での動作保証
- 商用 SaaS のような継続サポートや SLA

## 3. Public Beta のサポート境界

- サポートは `best effort` です。問い合わせは [GitHub Issues](https://github.com/aoyaman41/NewsVideo/issues) を前提にします
- API 利用料はユーザー自身の OpenAI / Google AI 契約に対して直接発生します
- AI モデル、生成品質、設定項目、既定値は今後変更される可能性があります
- 外部 API の仕様変更、障害、レート制限の影響を受けます
- 公開版を使う前に、このドキュメントと [クイックスタート](./クイックスタート.md) を確認してください

## 4. 既知の制約

### 配布とインストール

- 現在の配布フローでは `Developer ID` 署名と `notarization` を整備していません
- ローカル成果物の `NewsVideo.app` は `adhoc` 署名で、`spctl` 判定では reject される状態です
- そのため、配布版の扱いによっては macOS のセキュリティ警告や手動許可が必要になる可能性があります
- 実際の初回起動手順は [macOSインストールとGatekeeper](./macOSインストールとGatekeeper.md) を確認してください

### 機能面

- 画像スタイルは現在 `infographic` 固定です
- 既定の音声エンジンは `Gemini TTS` です
- 画像生成には `Google AI` が必要です
- 既定設定のまま使う場合、文章生成は `OpenAI` と `Google AI` の両方を前提にしています

### 実行環境

- AI 生成機能にはインターネット接続が必要です
- API キー未設定、クォータ超過、外部 API 側エラー時は生成処理が止まります
- 配布物には macOS 向けの `ffmpeg` / `ffprobe` バイナリを同梱しています

## 5. 使う前の判断ポイント

次に当てはまる場合は、現時点では見送ったほうが安全です。

- `Windows` または `Linux` で使いたい
- `Intel Mac` 向けの公式ビルドが必要
- notarization 済みの配布物でないと運用できない
- API キーを個別に管理したくない
- 生成品質や UI の変更が少ない安定版を求めている
