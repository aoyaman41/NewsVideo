# 対応環境と制約

このドキュメントは、2026-04-25 時点の `main` 実装と現在の配布物を基準に、NewsVideo の公式サポート範囲と Public Beta の制約を整理したものです。

## 1. 公式サポート範囲

### 配布版

| 項目 | 現在の扱い |
| --- | --- |
| OS | `macOS` |
| CPU | `Apple Silicon (arm64)` |
| 配布形式 | GitHub Releases の `.dmg` / `.zip` |
| サポートレベル | `Public Beta / best effort` |

現行の公開アセットは `NewsVideo-2.0.7-arm64.dmg` と `NewsVideo-2.0.7-arm64-mac.zip` を想定しています。`Intel Mac` 向け、`Windows` 向け、`Linux` 向けの公式配布物は現在ありません。

### ソースビルド

| 項目 | 現在の扱い |
| --- | --- |
| OS | `macOS` |
| Node.js | `^20.19.0` または `>=22.12.0` |
| 起動方法 | `npm install` → `npm run dev` |
| サポートレベル | `best effort` |

ソースビルド前提でも、現時点で積極的に案内しているのは `macOS` だけです。

## 2. 現在サポートしないもの

- `Intel Mac` 向けの公式配布物
- `Windows` 向けの公式配布物
- `Linux` 向けの公式配布物
- 非 macOS 環境での動作保証
- 商用 SaaS のような継続サポートや SLA
- 個別環境ごとの手厚いセットアップ支援
- 外部 API 障害やモデル仕様変更への即時追従保証
- 生成結果の品質保証や同一出力の再現保証

## 3. Public Beta のサポート境界

- サポートは `best effort` です。問い合わせは [GitHub Issues](https://github.com/aoyaman41/NewsVideo/issues) を前提にします
- まず自己解決を前提とし、環境構築や API アカウント設定の代行はしません
- API 利用料はユーザー自身の OpenAI / Google AI 契約に対して直接発生します
- AI モデル、生成品質、設定項目、既定値は今後変更される可能性があります
- 外部 API の仕様変更、障害、レート制限の影響を受けます
- 公開版を使う前に、このドキュメントと [クイックスタート](./クイックスタート.md) を確認してください

## 4. 既知の制約

### 配布とインストール

- 現在の配布フローでは `Developer ID` 署名と `notarization` を意図的に見送っています
- 理由は、個人開発 OSS の現状では Apple Developer Program の年額コストと運用負荷に見合わないためです
- ローカル成果物の `NewsVideo.app` は `adhoc` 署名で、`spctl` 判定では reject される状態です
- そのため、配布版の扱いによっては macOS のセキュリティ警告や手動許可が必要になる可能性があります
- 実際の初回起動手順は [macOSインストールとGatekeeper](./macOSインストールとGatekeeper.md) を確認してください

### 機能面

- 画像スタイルは `図解 / エディトリアル / ミニマル / SNS カード / 文字入りスライド / データカード` から選択できます
- 画像画面では、最大3枚のスライドサンプルをスタイル参照として画像生成に渡せます
- 画像生成バッチは一部失敗した場合に成功分を保存し、失敗したパートを画面に表示します
- 既定の音声エンジンは `Gemini TTS` です
- 既定の画像生成には `Google AI` が必要です。`GPT Image 2` を選ぶ場合は `OpenAI` キーを使います
- 既定設定のまま使う場合、文章生成は `OpenAI` と `Google AI` の両方を前提にしています

### 実行環境

- AI 生成機能にはインターネット接続が必要です
- API キー未設定、クォータ超過、外部 API 側エラー時は生成処理が止まります
- 配布物には macOS ネイティブ動画 helper を同梱しています
- `ffmpeg` / `ffprobe` は開発用 fallback としてのみ扱い、GitHub Releases 向け配布物には同梱しません

## 5. 使う前の判断ポイント

次に当てはまる場合は、現時点では見送ったほうが安全です。

- `Windows` または `Linux` で使いたい
- `Intel Mac` 向けの公式ビルドが必要
- `Developer ID` 署名や notarization 済みの配布物でないと運用できない
- API キーを個別に管理したくない
- API 障害やモデル変更に即応するサポートが必要
- 生成品質や動作を固定的に保証してほしい
- 生成品質や UI の変更が少ない安定版を求めている
