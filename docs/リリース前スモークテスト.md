# リリース前スモークテスト

このドキュメントは、Public Beta を GitHub Releases に出す前に、最低限そろっているべき動作確認を同じ観点で繰り返すためのチェックリストです。

## 目的

- 配布版の起動可否を確認する
- API キー保存と接続確認が通ることを確認する
- 記事入力から動画出力までの最短フローを確認する
- 保存 / 再読込の破綻がないことを確認する
- リリース可否を判断するための最低限の証跡を残す

## 前提条件

- 対象ビルドは `main` 由来の公開候補版
- 実行環境は `macOS / Apple Silicon (arm64)`
- ネットワーク接続あり
- API キーは以下のどちらかを用意済み

  - 既定設定のまま試す: `OpenAI` と `Google AI`
  - `Google AI` だけで試す: 事前に `設定 > 音声 > スクリプト生成モデル` と `設定 > 画像 > 画像プロンプト生成モデル` を両方 `Gemini 3.1 Pro` に変更

入力データ:

- [サンプル記事](./サンプル記事.md)
- 公開用の見本: [サンプル出力](./サンプル出力.md)

## 実施手順

### 1. 起動確認

- [ ] 配布版を起動して `プロジェクト` 画面が表示される
- [ ] クラッシュせずにサイドバーの `プロジェクト` と `設定` を往復できる

合格条件:

- 画面が白紙にならない
- 起動直後にブロッカーになるエラーダイアログが出ない

### 2. API キーと設定

- [ ] `設定` 画面で必要な API キーを保存できる
- [ ] 各サービスの `接続テスト` が成功する
- [ ] 設定画面上部のステータスが `保存済み` になる

合格条件:

- API キー保存後は伏せ字表示に戻る
- モデルを変更した場合も `保存済み` まで戻る

### 3. サンプル入力での最短フロー

- [ ] 新規プロジェクトを 1 件作成する
- [ ] `記事` 画面に [サンプル記事](./サンプル記事.md) のタイトル、出典、本文を貼り付ける
- [ ] 公開用サンプルと近い形で比較したい場合は `パート数` を `3` に変更する
- [ ] fresh start として `続きから自動生成` を実行する

合格条件:

- 記事の必須項目エラーが出ない
- `自動生成` カードのステータスが進む
- 途中で致命的なエラートーストが出ない

### 4. 生成完了の確認

- [ ] 自動生成完了後に `自動生成完了` の通知またはトーストが出る
- [ ] 上部の Workflow が最後まで進む
- [ ] `画像` 画面で各パートに画像が割り当てられている
- [ ] `音声` 画面で各パートに音声が割り当てられている
- [ ] `動画` 画面で `選択パートをプレビュー` と `動画を書き出し` が使える

合格条件:

- 生成内容の文言一致までは求めない
- 各ステップで最低 1 件以上の成果物ができ、画面遷移が継続できることを確認する

### 5. 出力と保存 / 再読込

- [ ] `動画` 画面でプレビューを確認する
- [ ] `.mp4` の書き出し先が表示される
- [ ] `Finderで表示` が使える、または出力先の `.mp4` が存在することを確認する
- [ ] アプリを閉じて再起動し、同じプロジェクトを `再開` できる
- [ ] 再読込後も記事タイトルと進捗、動画出力情報が残っている

合格条件:

- プロジェクト一覧に記事タイトルが表示される
- `進捗 5/5`、または少なくとも動画確認に戻れる状態が残る
- 再起動でプロジェクトが壊れない

## リリース判定

- 必須項目が 1 つでも落ちたら公開しない
- 外部 API 側の一時障害が疑われる場合でも、少なくとも 1 回は再試行して結果を残す
- 失敗がアプリ側か外部要因か判別できない場合は、Issue を起こしてから公開判断を行う

## 失敗時に残す情報

- 配布版のバージョン
- 実行した日時
- 使った API 構成 (`OpenAI + Google AI` / `Google AI only`)
- 失敗した画面と操作
- エラートーストまたは画面のスクリーンショット
- 生成途中まで進んでいる場合は、そのプロジェクト名
