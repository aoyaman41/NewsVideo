# 2. 実装計画書

## 2.1 プロジェクト概要

本ドキュメントは、報道動画自動生成アプリケーション「NewsVideo」の実装計画を定義する。

### 2.1.1 目的

テキスト記事から報道動画を自動生成するmacOSデスクトップアプリケーションを開発する。

### 2.1.2 スコープ

要件定義書に記載された全機能を実装する（フル機能）。

---

## 2.2 技術スタック

### 2.2.1 フレームワーク・言語

| カテゴリ | 技術 | バージョン | 選定理由 |
|----------|------|-----------|----------|
| フレームワーク | Electron | 最新安定版 | macOSネイティブ統合、ファイルシステムアクセス |
| フロントエンド | React | 18.x | コンポーネントベース開発、豊富なエコシステム |
| 言語 | TypeScript | 5.x | 型安全性、開発効率向上 |
| ビルドツール | Vite | 5.x | 高速なHMR、Electron統合が容易 |
| スタイリング | Tailwind CSS | 3.x | ユーティリティファースト、迅速なUI開発 |
| UIコンポーネント | shadcn/ui | 最新 | 高品質でカスタマイズ可能なコンポーネント |

### 2.2.2 外部サービス・API

> ※命名規約: 「0. 用語・モデル統一表.md」を参照

| 用途 | サービス | モデルID | コード内識別子 |
|------|----------|----------|---------------|
| スクリプト生成 | OpenAI API | gpt-5.2 | `openai` |
| 画像生成 | Google AI Studio | gemini-3-pro-image-preview (Nano Banana Pro) | `google_imagen` |
| 音声合成（クラウド） | Google Cloud TTS | chirp3-hd / gemini-tts | `google_tts` |
| 音声合成（ローカル） | macOS標準TTS | - | `macos_tts` |

### 2.2.3 主要ライブラリ

| カテゴリ | ライブラリ | 用途 |
|----------|-----------|------|
| 状態管理 | Zustand | グローバル状態管理 |
| ルーティング | React Router | ページ遷移 |
| フォーム | React Hook Form + Zod | フォーム管理・バリデーション |
| アニメーション | Framer Motion | UIアニメーション |
| 動画処理 | fluent-ffmpeg | ffmpegラッパー |
| 動画バイナリ | ffmpeg-static | ffmpegバイナリ同梱 |
| ストレージ | electron-store | 設定永続化 |
| 機密情報 | keytar | macOS Keychain連携 |
| ドキュメント処理 | mammoth | DOCX読み込み |
| 画像処理 | sharp | 画像リサイズ・変換 |
| HTTP通信 | axios | API通信 |
| ドラッグ&ドロップ | react-dropzone | ファイルドロップ |

## 2.6 リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| API料金の増大 | 中 | レート制限、キャッシュ活用、使用量モニタリング |
| ffmpeg処理の複雑さ | 中 | fluent-ffmpegで抽象化、テストケース充実 |
| 大容量ファイル処理 | 中 | ストリーミング処理、進捗表示、メモリ管理 |
| 外部API変更 | 低 | サービス層で抽象化、DI設計 |
| Electron更新による破壊的変更 | 低 | バージョン固定、段階的アップデート |
