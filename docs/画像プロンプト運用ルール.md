# 画像プロンプト運用ルール

このルールは、画像生成 preset の品質維持と実装ブレ防止を目的にします。

## 1. 対象と非対象

- 対象: `infographic` / `editorial` / `minimal` / `social` / `textRich` / `dataCard` の preset と `16:9` / `1:1` / `9:16` のアスペクト比。
- 非対象: 実写風、人物中心、自由入力の style prompt、多系統の完全カスタム運用。

## 2. 生成の2段階分離

- 段階A: 記事本文から抽出（JSON化）
- 段階B: 抽出結果から最終画像プロンプトを組み立て

この分離により、抽出責務と表現責務を分けます。

## 3. 抽出ガバナンス

- 抽出元は記事本文のみ。推測・補完・一般知識の混入を禁止。
- ノイズ文字列（パス、URL、ログ、コード断片）は無視。
- 人物・顔・番組名・ロゴ系は除外。
- 文字数制約:
- `topic` / `entities` / `locations` / `quantFacts` / `hero*` は短文制約
- `visualSlots.source` は短文制約
- `compositionNote` は120文字以内

## 4. 最終プロンプトガバナンス

- `infographic` / `editorial` / `minimal` / `social` は見出し/ラベル/数値中心。
- `textRich` / `dataCard` は、見出し、サブ見出し、短い要点、キー数値を画面内テキストとして扱ってよい。
- 画像内に出典表示は入れない。出典は動画の締めカード側で扱う。
- 人物・顔・ロゴ・透かし・番組名は禁止。
- プロンプト長上限: 通常は短く保つが、文字入りスライドや参照スタイル運用では構造化した指示を優先する。
- スタイルは preset から選び、自由入力の style prompt は通常 UI に出さない。
- レイアウトはアスペクト比ごとの既定パターンを使い、`16:9` 固定に戻さない。
- スタイル参照画像は最大3枚。参照画像の文字や内容をコピーせず、色・余白・文字階層・図形処理だけを合わせる。

## 5. バッチ生成戦略

- 画像プロンプト生成バッチは「単一抽出を最大10並列」で処理。
- 理由:
- 1回の巨大入力より、各パートの指示密度を高く保てる。
- 長文化を避け、抽出精度の劣化を抑えやすい。
- 画像生成バッチは最大3並列で処理し、一部失敗時は成功分を保存して失敗パートを明示する。
- 画像生成バッチはキャンセル要求を受けたら未開始タスクを進めない。

## 6. 変更時の必須確認

- `electron/ipc/ai.ts`
- `shared/project/imageStylePresets.ts`
- `runWithConcurrency(parts, 10, ...)`
- `createSinglePartExtractionPrompts(...)`
- `buildImagePromptText(...)`
- `MAX_IMAGE_PROMPT_CHARS`
- `electron/ipc/image.ts`
- `IMAGE_SYSTEM_POLICY_CORE`
- `IMAGE_BATCH_CONCURRENCY`
- `src/pages/ArticleInputPage.tsx` / `src/pages/ImageManagePage.tsx`
- `generateImagePrompts(parts, article, options)` に style / aspect ratio が渡っているか
- `generateImagePrompts(parts, article, options)` に styleReferenceImageIds / styleReferenceNote が渡っているか
- `image:generate` / `image:generateBatch` が styleReferenceImageIds を解決して画像生成 API に渡しているか

## 7. 変更してよい値

- 画像プロンプト抽出並列数（現在10）
- 画像生成並列数（現在3）
- 文字数上限
- ネガティブプロンプト語彙

変更する場合は、同コミットでこのファイルも更新してください。
