# 画像プロンプト運用ルール

このルールは、インフォグラフィック生成の品質維持と実装ブレ防止を目的にします。

## 1. 対象と非対象

- 対象: `infographic` 形式の 16:9 画像のみ。
- 非対象: 実写風、人物中心、サムネイル専用分岐、スタイル多系統運用。

## 2. 生成の2段階分離

- 段階A: 記事本文から抽出（JSON化）
- 段階B: 抽出結果から最終画像プロンプトを組み立て

この分離により、抽出責務と表現責務を分けます。

## 3. 抽出ガバナンス

- 抽出元は記事本文のみ。推測・補完・一般知識の混入を禁止。
- ノイズ文字列（パス、URL、ログ、コード断片）は無視。
- 人物・顔・番組名・ロゴ系は除外。
- 文字数制約:
- `topic` / `entities` / `locations` / `quantFacts` / `hero*` は短文制約
- `visualSlots.source` は短文制約
- `compositionNote` は120文字以内

## 4. 最終プロンプトガバナンス

- 長文禁止。見出し/ラベル/数値中心。
- 人物・顔・ロゴ・透かし・番組名は禁止。
- プロンプト長上限: 900文字。
- スタイルはフラットなベクター調のインフォグラフィック固定。

## 5. バッチ生成戦略

- バッチは「単一抽出を最大10並列」で処理。
- 理由:
- 1回の巨大入力より、各パートの指示密度を高く保てる。
- 長文化を避け、抽出精度の劣化を抑えやすい。

## 6. 変更時の必須確認

- `electron/ipc/ai.ts`
- `FIXED_IMAGE_STYLE_PRESET`
- `runWithConcurrency(parts, 10, ...)`
- `createSinglePartExtractionPrompts(...)`
- `buildImagePromptText(...)`
- `MAX_IMAGE_PROMPT_CHARS`
- `electron/ipc/image.ts`
- `IMAGE_SYSTEM_PROMPT_CORE`
- `src/pages/ArticleInputPage.tsx` / `src/pages/ImageManagePage.tsx`
- `generateImagePrompts(parts, article)` が style 引数なしで呼ばれているか

## 7. 変更してよい値

- 並列数（現在10）
- 文字数上限
- ネガティブプロンプト語彙

変更する場合は、同コミットでこのファイルも更新してください。
