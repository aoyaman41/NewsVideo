# 1. 要件定義書

## 1.1 システム概要

本システムは、macOS上で動作する報道動画自動生成デスクトップアプリケーションである。
テキスト記事を入力すると、以下を自動生成し、編集可能な形で提供する。

* 報道原稿（パート分割されたスクリプト）
* 報道動画用のスライド（静止画パネル）
* サムネイル画像
* オープニング・エンディングを含む最終動画

### 1.1.1 技術選定（正式決定）

**アプリケーションフレームワーク**: Electron + React + TypeScript

選定理由:
* Web技術による迅速なUI開発
* ffmpegとの統合が容易
* 豊富なUIライブラリエコシステム
* 将来的なWindows/Linux対応の可能性

### 1.1.2 外部AIサービス

| 用途 | サービス | API名称（コード内） | UI表示名 |
|------|----------|-------------------|----------|
| スクリプト生成 | OpenAI API | `openai` | OpenAI（スクリプト生成） |
| 画像生成 | Google AI Studio | `google_imagen` | Google AI（画像生成） |
| 音声合成 | Google Cloud TTS / macOS標準 | `google_tts` / `macos_tts` | Google TTS / macOS音声 |

**命名規約**:
* API名称（コード内）: スネークケース小文字（`google_imagen`）
* UI表示名: 日本語または正式名称
* 列挙値: スネークケース小文字（`generated`, `imported`）

## 1.2 用語定義

| 用語 | 定義 |
|------|------|
| 記事テキスト | 利用者が入力する元記事の本文テキスト |
| パート | 報道動画を構成する論点・段落単位のセクション |
| 報道原稿 / スクリプト | ナレーションとして読み上げる文章 |
| 画像プロンプト | 画像生成モデル向けのプロンプト・構成情報（JSON形式） |
| パネル画像 | 報道番組で用いられるスライド的な静止画 |
| サムネイル | 動画一覧に表示する代表画像 |
| プロジェクト | 記事・スクリプト・画像・動画設定をひとまとまりで管理する単位 |
| ジョブ | 非同期で実行される生成タスク（画像生成、音声生成等） |

| サービス | 必須/任意 | 取得先 |
|----------|----------|--------|
| OpenAI API | 必須 | https://platform.openai.com/ |
| Google AI Studio | 必須 | https://aistudio.google.com/ |
| Google Cloud TTS | 任意 | https://console.cloud.google.com/ |

### 1.4.3 ネットワーク要件

* AI API呼び出しにはインターネット接続が必要
* 動画レンダリングはローカルで実施（オフライン可）

## 1.5 機能要件

### 1.5.1 プロジェクト管理

#### 新規プロジェクト作成
* 記事テキストを入力またはファイルから読み込み
* 必要に応じて記事タイトルを入力
* 出典の入力（任意だが推奨 → 1.8.2参照）

#### プロジェクト保存 / 読み込み
* ローカルファイルとして保存（拡張子: `.newsproj`）
* 保存内容:
  * 記事テキスト
  * 分割されたパート情報
  * スクリプト（AI生成およびユーザー修正後）
  * 画像プロンプト
  * 生成済み画像のファイルパス・メタ情報
  * 動画出力設定
* 自動保存機能（3秒デバウンス）

#### プロジェクト一覧表示
* 最近使用したプロジェクトをリスト表示し、再編集可能
* 検索・フィルタ機能

### 1.5.2 記事入力・画像インポート

#### 記事テキスト入力
* テキストエリアへのペースト
* テキストファイル（.txt, .md, .docx）からの読み込み

#### 記事画像インポート
* 記事で使用されている画像をファイルとしてドラッグ&ドロップで登録
* 画像ごとにメモ・タグ（例: 「グラフ」「人物」「現場写真」）を付与可能
* 画像の著作権・利用条件メモ欄

#### 記事メタ情報
* 記事タイトル（必須）
* 出典URL/名称（推奨）
* 公開予定日（任意）

### 1.5.3 パート分割・スクリプト生成

#### パート分割
* 記事テキストから、ニュースとして自然な単位でパート分割案を生成
* 各パートに「パートタイトル」「要約」「原稿本文（読み上げ用）」を生成

#### スクリプト生成
* テレビ報道風の口調・フォーマットで原稿を生成
* 読み上げ時間を考慮した文字数に調整（1パートあたり30〜60秒、150〜300文字程度）

#### ユーザー編集
* 各パートのタイトル・本文を直接編集可能
* パートの追加・削除・並べ替えが可能
* **差分比較表示**: 再生成時に新旧スクリプトを比較表示し、採用/破棄を選択

### 1.5.4 画像プロンプト生成

#### 入力
* 記事テキスト全文
* 各パートのタイトル・スクリプト
* 利用者のスタイル指定（例: 「テレビ報道風」「落ち着いた配色」「文字なし」など）

#### 出力
* パネル画像用プロンプト（各パートごと）
  * 内容説明（prompt）
  * 不要要素（negative_prompt）
  * 構図・構成（人物の有無、テキスト表示有無など）
  * 想定サイズ・比率（16:9）
* サムネイル用プロンプト
  * 強調したいキーワード
  * 配色トーン
  * テキストの有無・配置方針

#### 形式
* JSON形式の構造化データ
* 完全再現用のseed値、生成パラメータを保持

#### ユーザー編集
* 画像プロンプトを表示・編集可能（上級者向け）

### 1.5.5 画像生成（Google AI）

#### API連携
* 画像プロンプトからAPIパラメータを構築
* aspectRatioと実サイズの対応:
  * 16:9 → 1920×1080
  * 1:1 → 1080×1080
  * 9:16 → 1080×1920（縦型動画用）

#### パネル画像生成
* 各パートごとに1枚以上のパネル画像を生成
* 生成枚数は設定で指定（パート1つにつき1〜3枚、デフォルト1枚）

#### サムネイル生成
* 画像プロンプトからサムネイルを生成

#### 記事画像の利用
* インポートされた記事画像を、パネル・サムネイル生成に組み込む指定が可能

#### 生成結果管理
* 生成された画像をプロジェクト内で一覧表示
* 各パートに割り当てる画像をユーザーが選択可能
* 不要な画像の削除
* プロンプト履歴とスタイルタグの表示

### 1.5.6 TTS による原稿読み上げ音声生成

#### 音声合成エンジン（抽象化レイヤー）

TTSサービスは抽象化レイヤーを通じて提供し、以下のエンジンを切り替え可能とする:

| エンジン | 特徴 | 要件 |
|----------|------|------|
| Google Cloud TTS | 高品質、多言語対応、SSML対応 | API Key必要 |
| macOS標準TTS | オフライン動作、無料 | macOS標準機能 |

#### 機能
* 各パートごとに原稿を音声化
* 言語・声質・話速・ピッチなどを設定画面またはプロジェクト設定で指定
* SSML対応（数値読み、ポーズ指定、固有名詞読み）
* 生成された音声をプロジェクトに保存し、パートに紐付け

### 1.5.7 動画構成・オープニング/エンディング設定

#### オープニング/エンディング動画設定
* ローカルファイル（mp4など）を設定画面で指定
* 長さ・解像度は読み取りのみ

#### パート動画構成
* 各パート:
  * 1枚以上のパネル画像
  * 対応する音声
* デフォルト挙動:
  * パートの音声再生中、割り当てられたパネル画像を一定時間で切り替え

#### 画像切り替えタイミング算出ルール

```
切り替え間隔 = 音声長(秒) / 画像枚数
各画像の表示時間:
  - 画像[0]: 0 〜 切り替え間隔
  - 画像[1]: 切り替え間隔 〜 切り替え間隔×2
  - ...
  - 画像[N-1]: 切り替え間隔×(N-1) 〜 音声長（端数吸収）
```

#### 全体構成
* オープニング動画
* パート1〜Nのスライド＋音声
* エンディング動画

#### 動画出力
* 出力フォーマット: MP4（H.264 / AAC）
* 解像度: 1920×1080（設定で変更可能: 1280×720, 3840×2160）
* フレームレート: 30fps（60fps選択可能）
* ハードウェアアクセラレーション: VideoToolbox利用（macOS）

### 1.5.8 プレビュー・編集・再生成

#### プレビュー機能
* 各パート単位のプレビュー（パネル＋音声）
* 全体動画の簡易プレビュー
* タイムラインに画像切替ポイントを表示

#### コメント入力による微調整
* 各パート／各画像／サムネイルに対して「コメント」欄を設置
* コメントをもとにGPTに再依頼:
  * スクリプト再生成
  * 画像プロンプト再生成
* **差分比較表示**: 再生成結果を適用前に新旧比較表示し、採用/破棄を選択

#### 手動編集
* スクリプト・画像割り当て・表示順をユーザーが手動で変更できる

### 1.5.9 設定画面・API Key管理

#### API Key設定
* OpenAI API Key
* Google AI Studio API Key
* Google Cloud TTS API Key（任意）

#### 保存
* macOS Keychainに安全に保存
* rendererプロセスにはキーを渡さず、mainプロセスからの代理呼び出し

#### 接続テスト
* API Key入力後、接続テストボタンで疎通確認
* レイテンシと結果を表示

#### 共通設定
* デフォルト動画解像度
* デフォルト音声設定（話速・声質・TTSエンジン選択）
* デフォルト画像スタイルプリセット
* オープニング／エンディング動画のパス設定
* 詳細ログ保存スイッチ

### 1.5.10 初回起動・自己診断

* 初回起動時に以下を自動チェック:
  * ffmpeg存在確認
  * Keychainアクセス確認
  * API接続テスト（設定済みの場合）
* 問題があれば設定ウィザードに誘導

## 1.6 非機能要件

### 1.6.1 パフォーマンス

| 項目 | 目標値 |
|------|--------|
| スクリプト生成 | 3000〜5000文字の記事で30秒以内 |
| 画像生成 | 1枚あたり30秒以内 |
| 音声生成 | 1パート（300文字）あたり10秒以内 |
| 動画レンダリング（FHD） | 10分動画で5分以内 |

### 1.6.2 信頼性

* プロジェクトデータはローカルに自動保存（オートセーブ）機能を持つ
* 自動保存間隔: 3秒（デバウンス）
* バックアップ: 保存時に前回バージョンを.bakとして保持

### 1.6.3 安全性

* API Keyは暗号化して保存し、ローカル外に漏えいしないよう配慮
* APIキーはmainプロセスのみが保持、rendererには渡さない
* エクスポート時に絶対パスを含めない（相対パスのみ）

### 1.6.4 ユーザビリティ

* ノンエンジニアでも直感的に操作できるUI
* キーボードショートカット対応
* アクセシビリティ: 色覚多様性配慮の配色プリセット

### 1.6.5 拡張性

* 他の画像生成モデル・TTSエンジンの追加が可能なアーキテクチャ
* サービス層は抽象化し、DIで差し替え可能

### 1.6.6 コスト管理

| サービス | 目安コスト（10分動画/5パート） |
|----------|------------------------------|
| OpenAI GPT-5.2 | 約$0.50〜$1.00 |
| Google AI（画像生成） | 約$0.20〜$0.50 |
| Google TTS | 約$0.05〜$0.10 |

* UI上に推定コスト表示機能
* 使用量モニタリング（日次/月次）
* レート制限時の待機・再試行ポリシー

### 1.6.7 ジョブキュー・再試行ポリシー

#### 同時実行制限
| ジョブ種別 | 同時実行数上限 |
|-----------|---------------|
| 画像生成 | 2 |
| 音声生成 | 3 |
| スクリプト生成 | 1 |

#### 再試行ポリシー
* 最大再試行回数: 3回
* バックオフ: 指数バックオフ（1秒、2秒、4秒）
* サーキットブレーカー: 5回連続失敗で一時停止、30秒後に再開

### 1.6.8 監視・可観測性

* 構造化ログ（JSON形式）
* ジョブ履歴の保持
* 失敗時の再実行メタデータ
* 「詳細ログ保存」スイッチを設定画面に配置

## 1.8 コンテンツポリシー

* インポート画像には利用条件メモ欄を設置
* 生成画像には透かし（SynthID）が含まれる旨を表示

### 1.8.2 ファクトチェック・ガードレール

報道用途のため、以下のガードレールを設ける:

* **出典必須の推奨**: 出典未入力時に警告表示
* **要約と原文の差分表示**: スクリプト生成後、元記事との対応を確認可能に
* **公開前チェックリスト**: 動画出力前に確認項目を表示
  * [ ] スクリプトの内容を確認した
  * [ ] 画像が記事内容と一致している
  * [ ] 出典を明記している
  * [ ] 著作権に問題がないことを確認した

### 1.8.3 モデレーション

* 生成コンテンツのセーフティフィルタ（Google AI標準）
* 不適切コンテンツ検出時の警告表示

## 1.9 想定ユースケース

1. ユーザーが記事テキストを貼り付けて「プロジェクト作成」
2. GPT-5.2でパート分割・スクリプト生成
3. 自動生成されたスクリプトをざっとチェックし、一部を修正
4. 画像プロンプトを自動生成し、Google AIでパネル画像・サムネイルを生成
5. 自動割り当てされた画像を確認し、「もう少し明るく」などコメントして一部再生成

7. 動画をプレビューし、パート順を微調整
8. 公開前チェックリストを確認
9. 最終動画を書き出し、YouTube等にアップロード

## 2.4 処理フロー・シーケンス

### 2.4.1 新規プロジェクト〜スクリプト生成

1. ユーザーが記事テキスト入力 → 「スクリプト生成開始」押下
2. ScriptGenerationService:
   * GPTにリクエスト
   * 出力JSONのバリデーション（必須キーチェック）
   * バリデーション失敗時はリカバリプロンプトで自動再試行
3. ProjectManager:
   * 生成結果をPartモデルにマッピング
4. UI:
   * パート編集画面に遷移

### 2.4.2 画像生成フロー

1. 「全パートの画像生成」押下
2. ImagePromptService:
   * GPTで各パートの画像プロンプト生成
3. JobQueueService:
   * 画像生成ジョブをキューに追加
   * 同時実行数: 2
4. ImageGenerationService:
   * Google AI APIに送信
   * レート制限時は指数バックオフで再試行
5. 進捗イベント:
   * IPC経由で`progress:update`を送信
6. UI:
   * 進捗モーダルで各プロンプトの状態を表示

### 2.4.3 コメントによる再生成フロー

1. ユーザーがコメント入力
2. CommentFeedbackService:
   * 既存プロンプト + コメントで修正版を生成
3. ImageGenerationService:
   * 修正版プロンプトで再生成
4. UI:
   * **差分比較表示**
   * 新旧を並べて表示、採用/破棄を選択

### 2.4.4 音声生成〜動画レンダリング

1. ユーザーが「全パート音声生成」押下
2. TTSService:
   * 選択されたエンジン（Google TTS / macOS）で音声生成
   * SSML対応（Google TTSの場合）
3. VideoRenderService:
   * ffmpegで動画レンダリング
   * VideoToolboxによるHWアクセラレーション（設定有効時）
4. 出力:
   * 指定パスにMP4を保存

## 2.6 設定保存・セキュリティ

* API Key:
  * macOS Keychainに保存
  * mainプロセスのみがアクセス
  * rendererにはキーを渡さない（代理呼び出し）

* プロジェクトファイル:
  * JSON形式で保存
  * 相対パスのみ使用（絶対パス禁止）
  * エクスポート時に個人情報を含まない

* Electronセキュリティ:
  * contextIsolation: true
  * nodeIntegration: false
  * webSecurity: true

## 2.7 エラー処理ポリシー

### 2.7.1 APIエラー

* レート制限: 指数バックオフで再試行（最大3回）
* タイムアウト: 再試行ボタン表示
* 認証エラー: 設定画面へ誘導

### 2.7.2 ジョブエラー

* サーキットブレーカー: 5回連続失敗で一時停止
* 再開: 30秒後に自動再開、または手動再開

### 2.7.3 データ整合性

* プロジェクト読み込み時、参照先ファイルが存在しない場合は警告表示
* ファイルバージョンが古い場合はマイグレーション実行
